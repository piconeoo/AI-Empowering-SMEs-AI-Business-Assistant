# 中小企业 AI 经营助理 Web MVP 设计规格

## 目标

构建一个本地可运行的 Web MVP，帮助中小企业老板和销售团队把客户需求、产品推荐、报价跟进、成交回款和经营日报串成一个轻量闭环。

第一阶段先交付可演示的产品骨架：仪表盘、客户管理、AI 销售助手，以及支撑这些页面的本地模拟 API 和业务逻辑。知识库、单据识别、日报通知在后续阶段逐步接入真实能力。

## 用户与场景

### 老板

老板关注每天经营状态：新客户、报价数、成交订单、待回款、待跟进客户、异常提醒。老板需要快速看到哪些客户和订单需要处理，而不是自己查 Excel 或追问销售。

### 销售

销售每天需要快速回复客户、推荐产品、整理报价说明、生成微信跟进话术、记录客户阶段。销售需要一个入口把“客户说了什么”和“下一步怎么跟”自动变成可执行动作。

## 第一阶段范围

### 包含

- 仪表盘：展示新客户、报价、成交、待回款、待跟进和异常提醒。
- 客户管理：展示客户列表、客户阶段、报价金额、跟进状态和负责人。
- AI 销售助手：输入客户需求后生成产品推荐、报价说明、微信跟进话术和跟进摘要。
- 本地模拟 API：提供仪表盘、客户列表和销售助手生成接口。
- 业务逻辑测试：覆盖经营指标汇总、客户阶段分组和销售助手文本生成。

### 不包含

- 正式登录、权限、多租户。
- 真实大模型 API 调用。
- 真实 OCR 和单据识别。
- 真实企业微信/钉钉通知。
- PostgreSQL、pgvector、MinIO 等生产级基础设施。

这些能力会在后续阶段补齐，第一阶段保留接口和页面位置。

## 产品结构

### 仪表盘

首页展示经营摘要，包含：

- 新客户数
- 报价数
- 成交订单数
- 待回款金额
- 待跟进客户数
- 异常提醒
- 今日重点行动建议

### 客户管理

客户列表展示：

- 客户名称
- 客户来源
- 客户阶段
- 最近需求
- 报价金额
- 回款状态
- 负责人
- 最近跟进时间

第一阶段数据来自本地模拟数据。

### AI 销售助手

销售输入客户需求，例如：

> 客户要一套适合小型工厂的进销存系统，预算 3 万以内，希望能支持微信通知和报价管理。

系统生成：

- 推荐产品
- 推荐理由
- 报价说明
- 微信跟进话术
- 跟进摘要
- 下一步动作

第一阶段使用规则和模板生成，后续替换为 LLM Provider。

## 技术架构

第一阶段采用单 Next.js 应用承载 UI 与本地 API：

- Next.js App Router 负责页面和 API routes。
- React 负责交互界面。
- CSS 负责第一版视觉样式。
- Node 内置测试框架验证核心业务逻辑。
- 本地 mock data 模拟客户、报价、订单、回款和提醒。

后续阶段会演进为：

- 前端：Next.js
- 后端：FastAPI
- 数据库：PostgreSQL + pgvector
- 文件存储：MinIO
- OCR：PaddleOCR
- 文档解析：MinerU
- AI 工作流：Dify 或自研 LLM Provider

## 错误处理

- API 参数为空时返回 400。
- 销售助手输入过短时返回可读错误。
- 页面加载失败时展示简洁错误状态。
- 第一阶段不做复杂重试，避免过度设计。

## 验收标准

- 可以启动本地 Web 应用。
- 首页能看到经营仪表盘。
- 客户管理能看到模拟客户列表。
- AI 销售助手能根据输入生成推荐、报价说明、微信话术和摘要。
- 核心业务逻辑测试通过。
- 阶段汇报说明已使用技术、已实现功能、运行方式和下一阶段计划。
